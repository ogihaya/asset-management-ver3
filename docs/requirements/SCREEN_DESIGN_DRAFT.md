# 画面設計ドラフト

本ドラフトは `docs/requirements/public/screen_design_png` 配下の画像をもとに、既存の `OVERVIEW.md` と `REQUIREMENTS.md` の要件に沿って画面情報を文章化したものである。画像から読み取れない詳細な挙動、入力バリデーション、エラーメッセージ文言などは本ドラフトでは未確定のものもあるが、追加要件として指定されたモバイル時の上部ヘッダー、各入力項目の編集モーダル、月次確定時の支出確認モーダルは本ドラフトで要件として確定する。

## ログイン画面

### 画面概要
- メールアドレスとパスワードでログインするための画面とする。
- ログイン画面の見た目に強い制約は設けず、既存の認証要件を満たせていればよい。
- ログインに失敗した場合やセッション失効時に、理由を表示して再入力できる前提とする。

### 主な表示要素
- メールアドレス入力欄
- パスワード入力欄
- ログインボタン
- パスワード再設定画面への導線
- 新規登録画面への導線
- ログイン失敗またはセッション失効理由の表示領域

### 主な操作
- メールアドレスとパスワードを入力してログインする。
- パスワードを忘れた場合は再設定導線へ遷移する。
- 未登録ユーザーは新規登録導線へ遷移する。

### 補足
- 画像は存在しないため、UIレイアウトの詳細は未確定とする。

### レスポンシブ方針
- PCとスマホの両方で利用可能とする。
- 具体的なモバイルレイアウトは別途詳細化する。

## ダッシュボード画面

### 画面概要
- ログイン後に最初に確認する画面とし、資産状況と投資判断に必要な情報を一覧できる構成とする。
- 画面左側に共通サイドバー、右側にメインコンテンツを配置する。
- メインコンテンツは、PCでは「資産サマリーカード」と「投資構成カード」を横並びにし、その下に「将来収支と投資判断カード」を配置する。さらにその下にグラフカードを配置する。

### 主な表示要素
- 左サイドバー(可変、閉じることもできる)
    - アプリ名表示(資産管理)
    - 画面遷移メニュー
        - `ダッシュボード`
        - `月次記録`
        - `ライフプラン`
        - `設定`
    - ログアウト導線(一番下に配置)
- 最新の資産サマリーカード
    - 最新の確定済み月を基準に、総資産、収入、支出などの主要指標を表示する領域
    - 通常資産のうち、評価額の絶対値が大きい上位3件を表示し、各資産の名称、評価額、総資産比率を確認できるようにする
    - 通常資産全件を確認するモーダル導線
- 将来収支と投資判断カード
    - 投資可能額や採用された制約イベントなど、投資判断に必要なテキスト情報を表示する領域
    - 予測月次収入、予測月次支出、平均対象月数などの将来収支予測情報
- 投資構成カード
    - 最新確定月の投資評価額構成比を表す円グラフ
    - 各投資先の実績構成比、理想比率、差分、最新確定月時点の評価額一覧
- 資産グラフカード
    - 資産選択プルダウン
    - 表示期間指定UI
    - 資産推移を表示する折れ線グラフ

### 主な操作
- サイドバーから各画面へ遷移する。
- ログアウト導線からログアウトする。
- 資産選択プルダウンで表示対象の資産を切り替える。
- 期間指定UIでグラフの表示期間を切り替える。

### 補足
- 最新の資産サマリーカードは、常に最新の確定済み月を基準に表示し、未確定月の入力途中データはここでは表示しない。
- 通常資産一覧は投資資産を含めず、表示対象となっている通常資産を評価額の絶対値が大きい順で表示する。
- 資産サマリーカードでは上位3件のみを常時表示し、残りの通常資産はモーダルを開いて確認する。
- 将来収支と投資判断カードには、既存要件に従い `投資可能額` や `配分概要` などの情報を表示する想定とする。投資可能額は複数ライフイベントを比較し、最も金額が低くなるイベントを基準に表示する。
- 将来収支予測は、確定済み月の直近6か月平均から自動算出した `予測月次収入 / 予測月次支出` を表示し、平均対象月数も確認できるようにする。投資構成カードの円グラフは理想比率ではなく、最新確定月の投資評価額構成比を表示する。
- グラフの表示対象は既存要件に従い単一対象切替方式とし、期間指定は `3か月 / 6か月 / 1年 / 全期間` を想定する。表示対象には通常資産に加えて投資先ごとの評価額推移も含める。

### レスポンシブ方針
- PCでは左サイドバーを表示する。
- PCでは `lg` 以上の幅で「資産サマリーカード」と「投資構成カード」を横並びにし、その下に「将来収支と投資判断カード」を独立配置する。
- モバイルでは左サイドバーの代わりに、画面上部に共通ヘッダーを表示する。
- 共通ヘッダーには `ダッシュボード / 月次記録 / ライフプラン / 設定` へ遷移するアイコンを横並びで配置し、右端にユーザーアイコンを配置する。
- ユーザーアイコン押下時はユーザーメニューを表示し、ユーザー情報確認やログアウト導線にアクセスできるようにする。
- サマリーカード、将来収支と投資判断カード、投資構成カード、資産グラフカードはモバイル時は基本的に縦積みで表示する。

## 月次記録画面

### 画面概要
- 月ごとの資産、収入、投資情報を入力し、月次確定を行うための画面とする。
- 画面上部に対象月の表示領域を置き、その下に入力カードを配置する。
- メイン領域は `資産入力カード`、`収入入力カード`、`投資入力カード` の3つで構成する。

### 主な表示要素
- 左サイドバー
    - アプリ名表示
    - 画面遷移メニュー
        - `ダッシュボード`
        - `月次記録`
        - `ライフプラン`
        - `設定`
    - ログアウト導線
- 表示対象月切替領域
    - 前月へ移動するボタン
    - 年月表示
    - 翌月へ移動するボタン
- 資産入力カード
    - カード追加ボタン
    - 資産名の一覧
    - 各項目の金額入力欄
    - 各項目右側の編集アイコン
- 収入入力カード
    - カード追加ボタン
    - 収入明細名の一覧
    - 各項目の金額入力欄
    - 各項目右側の編集アイコン
- 投資入力カード
    - 設定画面で定義された投資先名の一覧
    - 各投資先のその月の評価額入力欄
- 月次確定ボタン
- 月次確定確認モーダル
    - 支出自動推定値の表示
    - 支出補正入力欄
    - 推定不可時の理由表示領域
    - 空欄がある場合の警告表示領域
    - キャンセルボタン
    - 確定ボタン
- 資産追加モーダル
    - 資産名入力欄
    - 追加ボタン
- 資産編集モーダル
    - 資産名入力欄
    - 金額入力欄
    - 削除ボタン
    - 保存ボタン
- 収入追加モーダル
    - 収入名入力欄
    - 追加ボタン
- 収入編集モーダル
    - 収入名入力欄
    - 金額入力欄
    - 削除ボタン
    - 保存ボタン

### 主な操作
- 表示対象月を確認し、その月の記録を入力する。
- 前月ボタンまたは翌月ボタンで表示対象月を切り替える。
- 資産入力カードで各資産の当月金額を入力する。
- 資産入力カードの追加ボタンを押すと、資産追加モーダルを表示する。
- 資産追加モーダルでは資産名のみを入力し、追加後は一覧へ行追加して当月金額を inline で入力する。
- 資産の編集アイコンを押すと、未確定月に限り資産名と当月金額を編集できるモーダルを表示する。
- 資産削除を選択した場合は確認ダイアログを表示したうえで削除し、削除対象の資産は次の未確定月から非表示にする。
- 収入入力カードで各収入明細の当月金額を入力する。
- 収入入力カードの追加ボタンを押すと、収入追加モーダルを表示する。
- 収入追加モーダルでは収入名のみを入力し、追加後は一覧へ行追加して当月金額を inline で入力する。
- 収入の編集アイコンを押すと、該当明細の編集または削除を行うモーダルを表示する。
- 収入削除を選択した場合は、確認ダイアログを表示したうえで削除を実行する。
- 投資入力カードでは、設定画面で定義済みの投資先についてその月の評価額のみを inline で入力する。
- 月次記録画面では投資先の追加、編集、削除は行わない。
- 入力完了後に月次確定ボタンを押すと、支出自動推定値と補正入力欄を表示するモーダルを開く。
- 支出補正が不要な場合は、そのまま確定ボタンを押して対象月を確定する。
- 支出補正が必要な場合は、モーダル内で補正値を入力したうえで確定する。
- 資産、収入、投資評価額のいずれかに空欄がある場合は、警告を表示して確定不可とする。

### 補足
- 資産入力カードは一覧形式で、左列に名称、右列に金額を入力する構成とする。
- 資産追加時は資産名のみを登録し、名称登録後に当月金額を入力する。
- 資産名を変更した場合でも、過去の確定月に保存済みの名称は変更しない。
- 収入入力カードは明細単位の追加を前提とし、収入名は翌月以降も持ち越して再利用できる前提とする。
- 投資入力カードは設定画面で定義した投資先を表示し、月次記録ではその月の評価額の入力だけを行う構成とする。
- 支出自動推定値と支出補正UIは、月次確定ボタン押下後に表示するモーダル内で扱う。
- 初月など支出推定が計算不可の月は、モーダル内で推定不可理由を表示する前提とする。
- 月次確定は既存要件に従い手動操作で行い、古い月から順番にのみ確定可能とする。
- 月次確定前に、その月に表示されている資産、収入、投資評価額がすべて入力済みである必要がある。
- 月次確定モーダルは支出確認と補正のみを扱い、投資計算結果はこのモーダル内には表示しない。

### レスポンシブ方針
- スマホ対応は必須とする。
- モバイルでは左サイドバーの代わりに、画面上部に共通ヘッダーを表示する。
- 共通ヘッダーには `ダッシュボード / 月次記録 / ライフプラン / 設定` のアイコンを横並びで配置し、右端にユーザーアイコンを置く。
- ユーザーアイコン押下時はユーザーメニューを表示し、ユーザー情報確認やログアウト導線にアクセスできるようにする。
- `資産入力カード / 収入入力カード / 投資入力カード` はモバイル時は基本的に縦積みで表示する。

## ライフプラン画面

### 画面概要
- 将来の支出イベントを一覧で管理するための画面とする。
- 画面右上にライフプラン追加ボタン(常に表示)を配置し、中央にイベント一覧を表示する構成とする。

### 主な表示要素
- 左サイドバー
    - アプリ名表示
    - 画面遷移メニュー
        - `ダッシュボード`
        - `月次記録`
        - `ライフプラン`
        - `設定`
    - ログアウト導線
- ライフプラン追加ボタン
- ライフプランイベント一覧
    - 年月表示
    - イベント内容表示
    - 金額表示
    - メモ表示
    - 各項目右側の編集アイコン
- ライフプラン追加モーダル
    - 年月入力欄
    - イベント内容入力欄
    - 金額入力欄
    - メモ入力欄
    - 追加ボタン
- ライフプラン編集モーダル
    - 年月入力欄
    - イベント内容入力欄
    - 金額入力欄
    - メモ入力欄
    - 削除ボタン
    - 保存ボタン

### 主な操作
- ライフプラン追加ボタンから新しい将来イベントを登録する。
- 一覧で登録済みイベントの年月、内容、金額、メモを確認する。
- 各イベント右側の編集アイコンを押すと、該当イベントの編集または削除を行うモーダルを表示する。
- 削除を選択した場合は、確認ダイアログを表示したうえで削除を実行する。

### 補足
- 画像上では一覧は単純な行形式で描かれており、各行は `年月 / 内容 / 金額` の3要素で構成される。
- 画面設計上は、一覧にメモも常時表示する。
- 同じ年月に複数イベントがある場合も、既存要件に従いイベント単位で別行として保持する前提とする。
- 編集と削除は、各イベント右側の編集アイコンから開くモーダルで行う。

### レスポンシブ方針
- スマホでも閲覧・登録可能とする。
- モバイルでは左サイドバーの代わりに、画面上部に共通ヘッダーを表示する。
- 共通ヘッダーには `ダッシュボード / 月次記録 / ライフプラン / 設定` のアイコンを横並びで配置し、右端にユーザーアイコンを置く。
- ユーザーアイコン押下時はユーザーメニューを表示し、ユーザー情報確認やログアウト導線にアクセスできるようにする。
- 一覧はモバイル時に見やすいよう縦方向を優先した構成とし、必要に応じて各イベント情報を縦積みで表示する。

## 設定画面

### 画面概要
- 投資計算に使う設定値を編集し、保存するための画面とする。
- 左側に共通ナビゲーション、右側のメイン領域に設定カードを並べる構成とする。
- 主要な設定対象は `生活防衛資金`、`投資先配分比率`、`将来収支予測` の3つとする。

### 主な表示要素
- 左サイドバー
    - アプリ名またはロゴ表示
    - 画面遷移メニュー
        - `ダッシュボード`
        - `月次記録`
        - `ライフプラン`
        - `設定`
    - ログアウト導線
- 生活防衛資金カード
    - 防衛資金額入力欄
    - 入力欄右側の編集アイコン
    - 保存ボタン
- 投資先配分比率カード
    - 投資先一覧編集ボタン
    - 投資先名の一覧
    - 比率入力または表示欄
    - 合計比率表示
    - 残り比率表示
- 将来収支予測カード
    - 予測月次収入表示
    - 予測月次支出表示
    - 計算対象月数表示
    - 予測方式説明（確定済み直近6か月平均）
    - 予測ルール説明（賞与・臨時収入を含む、ライフプランイベントは別加算）
- 生活防衛資金編集モーダル
    - 防衛資金額入力欄
    - 保存ボタン
- 投資先一覧編集モーダル
    - 投資先名入力欄の行一覧
    - 比率入力欄の行一覧
    - 行追加ボタン
    - 行削除ボタン
    - 合計比率表示
    - 残り比率表示
    - 保存ボタン

### 主な操作
- 生活防衛資金カードはカード単位で個別保存する。
- 投資先配分比率カードの変更は、一覧一括編集モーダル単位で保存する。
- 生活防衛資金入力欄右側の編集アイコンを押すと、生活防衛資金の編集モーダルを表示する。
- 生活防衛資金の変更は、生活防衛資金カード内の保存ボタンで保存する。
- 投資先一覧編集ボタンを押すと、投資先の追加・編集・削除をまとめて行える一覧一括編集モーダルを表示する。
- 一覧一括編集モーダルでは、投資先名と比率を全行まとめて調整しながら新規追加できる。
- 一覧一括編集モーダルでは、合計比率と残り比率を確認しながら調整できる。
- 投資先配分の変更は、一覧一括編集モーダル内の保存ボタンでまとめて保存する。
- 将来収支予測カードでは、投資計算に使う自動予測値を確認する。
- 将来収支予測は手動入力せず、確定済み月の直近6か月平均から自動算出する。

### 補足
- 投資先配分比率カードは、投資先名と比率の一覧表として表現する。
- 合計比率は既存要件に従い 100% である必要があるため、一覧一括編集モーダル内で合計表示と残り比率表示を確認できるようにする。
- 設定変更時の再計算対象は、既存要件に従い未確定月のみとする。
- 将来収支予測カードは確認用カードであり、入力欄や保存操作は持たない。
- 収入予測には賞与や臨時収入も含め、将来の大きな支出はライフプランイベントとして通常支出と別に扱う。
- サイドバーの見た目やアイコン、ロゴ表現は画像に依存する参考情報であり、今後のモックアップ作成には参考にせず１から考えて作り、機能要件としては主要導線が存在することを重視する。
- 生活防衛資金はカード単位で保存し、投資先配分は一覧一括編集モーダル単位で保存する。
- 月次記録画面に表示する投資先の一覧は、この設定画面で保存された投資先配分を基準にする。

### レスポンシブ方針
- スマホ対応は必須とする。
- モバイルでは左サイドバーの代わりに、画面上部に共通ヘッダーを表示する。
- 共通ヘッダーには `ダッシュボード / 月次記録 / ライフプラン / 設定` のアイコンを横並びで配置し、右端にユーザーアイコンを置く。
- ユーザーアイコン押下時はユーザーメニューを表示し、ユーザー情報確認やログアウト導線にアクセスできるようにする。
- 生活防衛資金カードと投資先配分比率カードはモバイル時は縦積みで表示する。
