# 機能要件書

## 1. システム概要
本システムは、ユーザーが月次で資産・収入を記録し、確定済み実績から自動予測した将来収支と将来のライフプラン支出を踏まえた投資判断を支援するWebアプリケーションである。  
目的は、現状資産の把握を容易にし、「長期・積立・分散」の三原則を守った投資可能額と投資先配分を算出することで将来資産の最大化を支援することにある。

v1では、メール+パスワード認証、月次記録、支出推定と補正、月次確定、投資計算、可視化を対象とする。  
v1対象外は、月次リマインド通知、データエクスポート、バックアップ、外部サービス自動連携とする。

## 3. 機能要件
### 3.1 認証・アカウント管理
#### FR-001: ユーザー自己登録
- **説明**: ユーザーはメールアドレスとパスワードで自己登録できる。
- **入力**: メールアドレス、パスワード。
- **出力**: 登録完了状態、ユーザーアカウント情報。
- **ビジネスルール**:
- パスワードは8文字以上かつ英数字混在とする。
- 既存メールアドレスでの重複登録は不可とする。
- **受入基準**:
- 新規メールアドレスで登録できること。
- 既存メールアドレスで登録を試行した場合にエラーになること。
- パスワード要件を満たさない場合にエラーになること。

#### FR-002: ログイン・セッション管理
- **説明**: ユーザーは登録済み認証情報でログインし、セッションを保持できる。
- **入力**: メールアドレス、パスワード。
- **出力**: ログイン成功/失敗、認証済みセッション。
- **ビジネスルール**:
- セッション有効期限は30日とする。
- 複数端末からの同時ログインを許可する。
- ユーザーごとにデータを分離する。
- セッション失効時は再ログイン画面へ遷移し、失効理由を表示する。
- **受入基準**:
- 正しい認証情報でログインできること。
- 誤った認証情報でログイン失敗すること。
- 2端末以上で同時ログインして利用できること。
- 別ユーザーのデータが参照できないこと。
- セッション失効時に再ログイン画面へ遷移し、理由文言が表示されること。

#### FR-003: パスワード再設定
- **説明**: ユーザーはメール経由でパスワードを再設定できる。
- **入力**: 登録メールアドレス、新パスワード。
- **出力**: 再設定完了状態。
- **ビジネスルール**:
- 再設定後のパスワードはFR-001の強度要件を満たすこと。
- **受入基準**:
- 登録済みメールアドレスで再設定フローを開始できること。
- 再設定後のパスワードでログインできること。

#### FR-004: 退会（論理削除）
- **説明**: ユーザーは退会できる。退会は論理削除で扱う。
- **入力**: 退会実行操作。
- **出力**: 退会済み状態。
- **ビジネスルール**:
- 退会後はログイン不可とする。
- データは無期限保持とする。
- 論理削除されたアカウントの復旧は運用者のみ可能とする。
- **受入基準**:
- 退会後にログイン不可となること。
- 退会アカウントが論理削除状態で保持されること。
- 運用者操作でのみ復旧可能なこと。

### 3.2 月次データ管理
#### FR-101: 月次資産入力
- **説明**: ユーザーは月次で資産を名称単位に入力できる。
- **入力**: 月（JST）、名称、評価額。
- **出力**: 月次資産明細（当月の入力対象資産）、月次資産合計。
- **ビジネスルール**:
- 必須項目は「名称+評価額」。
- 同一ユーザー内で資産名の重複は禁止。
- 資産はカテゴリではなく名称単位で管理する。
- 月次入力対象は、当月時点で有効な資産一覧（FR-106）とする。
- 負数入力を許可し、クレジットカード引落予定額などの負債表現にも利用できるようにする。
- **受入基準**:
- 必須項目が揃っていれば登録できること。
- 同一ユーザー内で同名資産の重複登録が拒否されること。
- 負数を含む資産金額を登録できること。

#### FR-102: 月次収入入力（複数明細）
- **説明**: ユーザーは月次収入を複数明細で記録できる。
- **入力**: 月（JST）、収入明細（項目名、金額、メモ任意）。
- **出力**: 月次収入明細、月次収入合計。
- **ビジネスルール**:
- 収入は複数明細を登録可能とする。
- 収入明細名は翌月以降も再利用可能とする。
- **受入基準**:
- 1か月に複数件の収入明細を登録できること。
- 明細合計が月次収入合計として算出されること。
- 過去に使用した収入明細名を翌月以降でも再利用できること。

#### FR-103: 支出自動推定と補正
- **説明**: システムは支出を自動推定し、ユーザーは確定前に補正できる。
- **入力**: 前月総資産、当月収入合計、当月総資産、補正値（任意）。
- **出力**: 推定支出、確定前支出値。
- **ビジネスルール**:
- 支出推定式は `前月総資産 + 当月収入 - 当月総資産` とする。
- 推定値は月次確定前のみ手動補正可能とする。
- 初月（前月総資産が存在しない月）は支出推定を計算不可とする。
- **受入基準**:
- 式に基づき推定支出が算出されること。
- 確定前は補正入力が保存できること。
- 確定後は補正できないこと。
- 初月は支出推定結果を表示せず、計算不可理由が表示されること。

#### FR-104: 月次確定とロック
- **説明**: ユーザーは月次データを手動確定でき、確定後は編集不可となる。
- **入力**: 確定対象月、確定実行操作。
- **出力**: 月次確定状態、ロック状態。
- **ビジネスルール**:
- 月次確定は古い月から順番にのみ可能。
- 月次確定期限は翌月末。
- 月次確定実行時は、支出自動推定値と補正入力欄を表示する確認モーダルを経由する。
- 表示中の通常資産、収入、投資評価額に空欄がある月は確定不可とする。
- 期限超過月は入力可能だが「対象月一覧＋警告バナー」で表示する。
- 確定済み月の再オープンは不可。
- **受入基準**:
- 未確定の古い月を飛ばして新しい月を確定できないこと。
- 確定操作時に支出自動推定値または計算不可理由が表示されること。
- 補正値を入力したうえで確定できること。
- 空欄がある場合は警告が表示され、確定できないこと。
- 確定後は当該月データを編集できないこと。
- 期限超過月で対象月一覧と警告バナーが表示されること。

#### FR-105: 未確定月の遡り入力
- **説明**: 未確定月に対しては過去月でも入力できる。
- **入力**: 未確定の過去月データ。
- **出力**: 更新された未確定月データ。
- **ビジネスルール**:
- JST月初〜月末を月次境界とする。
- 未確定月のみ遡り入力可。
- **受入基準**:
- 未確定月に遡って入力できること。
- 確定月では遡り入力できないこと。

#### FR-106: 資産ライフサイクル管理
- **説明**: 資産を追加・編集・削除・復活し、月次入力対象として継続管理する。
- **入力**: 資産追加操作、資産編集操作、資産削除操作、資産復活操作、対象月、最新の確定月。
- **出力**: 月次入力対象資産一覧、更新後の資産状態。
- **ビジネスルール**:
- 資産追加時は名称のみを登録し、その後に当月金額を入力する。
- 追加した資産は翌月以降も月次入力対象として継続表示する。
- 翌月以降に自動表示される資産の評価額初期値は空欄とする。
- 資産名と金額の編集は未確定月に限り可能とする。
- 未確定月で資産名を変更しても、過去の確定月に保存済みの名称は変更しない。
- 資産削除時は対象資産を削除し、最新の確定月の次の未確定月から非表示にする。
- 削除済み資産は復活可能とし、復活時は最新の確定月の次の未確定月から再表示する。
- 復活時も削除期間の月は非表示のままとする。
- **受入基準**:
- 資産名のみで新規資産を追加できること。
- 追加した資産が翌月以降の月次入力対象にも表示されること。
- 翌月以降に自動表示された資産の評価額が空欄で表示されること。
- 未確定月では資産名と当月金額を編集できること。
- 過去の確定月データの名称が、未確定月での名称変更に影響されないこと。
- 資産削除後、最新の確定月までは履歴として保持され、次の未確定月から当該資産が非表示になること。
- 復活後、最新の確定月の次の未確定月から当該資産が再表示されること。
- 削除期間の月は復活後も入力項目に表示されないこと。

#### FR-107: 月次投資評価額入力
- **説明**: ユーザーは月次記録画面で、設定済み投資先に対するその月の評価額を入力できる。
- **入力**: 月（JST）、設定済み投資先一覧、各投資先の評価額。
- **出力**: 月次投資評価額明細。
- **ビジネスルール**:
- 月次記録画面に表示する投資先一覧は、設定画面で保存された投資先配分を基準とする。
- 月次記録画面では投資先の追加、編集、削除は行わない。
- 月次記録画面では各投資先のその月の評価額のみを入力対象とする。
- 投資評価額は通常資産入力とは別管理とし、総資産集計に含める。
- 投資評価額は0以上とする。
- **受入基準**:
- 設定済み投資先が月次記録画面に一覧表示されること。
- 各投資先のその月の評価額のみを入力できること。
- 月次記録画面に投資先の追加、編集、削除導線が存在しないこと。
- 負数の投資評価額を入力できないこと。

### 3.3 ライフプラン・投資計算
#### FR-201: ライフプランイベント登録
- **説明**: 将来支出を単発イベントとして登録できる。
- **入力**: 年月、金額、メモ。
- **出力**: ライフプランイベント一覧。
- **ビジネスルール**:
- イベント時点は年月粒度で管理。
- 同年月の複数イベントはイベント単位で別管理する。
- **受入基準**:
- 年月・金額を指定してイベント登録できること。
- 登録イベントが一覧参照できること。
- 同年月に複数イベントを登録した場合、個別イベントとして保持・参照できること。

#### FR-202: 生活防衛資金設定
- **説明**: 投資計算に使用する生活防衛資金を設定できる。
- **入力**: 防衛資金額。
- **出力**: 現在有効な防衛資金額。
- **ビジネスルール**:
- 初回入力は必須とする。
- 2回目以降は前回値を引き継ぎ、必要時に変更可能とする。
- **受入基準**:
- 初回未入力時は投資計算できないこと。
- 設定済みの場合は再入力なしで計算できること。

#### FR-203: 投資計算の実行条件判定
- **説明**: 投資計算前提が不足している場合は計算不可とし、不足項目を明示する。
- **入力**: 月次確定済みデータ、将来収支予測、ライフプラン、防衛資金、投資先配分。
- **出力**: 計算可否、不足項目一覧。
- **ビジネスルール**:
- ライフプラン未登録時は計算不可。
- 将来収支予測を算出できない場合は計算不可。
- 計算不可時は不足項目を箇条書きで表示する。
- **受入基準**:
- 必要前提不足時に結果を表示せず不足項目が表示されること。
- 前提が揃った場合のみ計算できること。
- 不足項目が箇条書きで表示されること。

#### FR-203A: 将来収支自動予測
- **説明**: システムは確定済み実績から、投資計算に使う予測月次収入と予測月次支出を自動算出する。
- **入力**: 確定済み月の収入実績、確定済み月の支出実績、対象月。
- **出力**: 予測月次収入、予測月次支出、平均対象月数。
- **ビジネスルール**:
- 予測対象に使う履歴は、対象月以前の確定済み月のみとする。
- 予測方式は確定済み直近6か月平均とする。
- 確定済み月が6か月未満の場合は、存在する確定済み月のみを平均対象とする。
- 収入予測には賞与・臨時収入を含める。
- 支出予測は確定済み月の支出実績をそのまま平均し、過去の突発支出を自動除外しない。
- 将来の大きな支出はライフプランイベントで別管理し、予測月次支出には含めない。
- 将来収支予測の手動入力および手動補正は行わない。
- **受入基準**:
- 確定済み月が6か月以上ある場合、直近6か月平均で予測値が算出されること。
- 確定済み月が1〜5か月の場合、存在する確定済み月のみで予測値が算出されること。
- 収入予測に賞与や臨時収入が含まれること。
- ライフプランイベントが予測月次支出には含まれず、別管理されること。
- 将来収支予測の手動入力欄や手動補正導線が存在しないこと。

#### FR-204: 投資可能額計算
- **説明**: 月次確定時に投資可能額を算出する。
- **入力**: 予測月次収入、予測月次支出、ライフプランイベント、防衛資金、対象月までの月数。
- **出力**: 投資可能額。
- **ビジネスルール**:
- 各ライフプランイベントについて `累積収入 = 予測月次収入 × 現在月から対象月までの月数` を算出する。
- 各ライフプランイベントについて `累積支出 = 予測月次支出 × 現在月から対象月までの月数 + 対象イベント月までのライフプラン支出合計` を算出する。
- `余力資産 = 累積収入 - 累積支出 - 生活防衛資金`。
- 各ライフプランイベントについて `投資可能額 = 余力資産 ÷ 現在月から対象月までの月数` を算出する。
- 複数イベントの投資可能額を比較し、最も金額が低いイベントの値を採用する。
- 最小値が同額で並ぶ場合は、より早い年月のイベントを採用する。
- 投資可能額が0未満の場合は0円として扱う。
- 計算実行タイミングは月次確定時の自動実行。
- 投資先比率や生活防衛資金などの設定変更時、再計算対象は未確定月のみとする。
- 提案スタンスは中立とし、式ベースで機械的に計算する。
- **受入基準**:
- ルール通りに投資可能額が算出されること。
- 複数ライフプランイベントがある場合、最も金額が低いイベントの投資可能額が採用されること。
- 同額時はより早い年月のイベントが採用されること。
- 負値時に0円になること。
- 月次確定時に自動実行されること。
- 累積収入と累積支出が予測月次収入・予測月次支出とライフプランイベントから算出されること。

#### FR-205: 投資先配分計算
- **説明**: ユーザー定義投資先へ比率に基づき投資可能額を配分する。
- **入力**: 投資可能額、投資先一覧、投資先比率。
- **出力**: 投資先ごとの配分額。
- **ビジネスルール**:
- 投資先の追加、編集、削除および比率設定は設定画面で行う。
- 設定画面では投資先の追加、編集、削除を一覧一括編集モーダルでまとめて行えること。
- 月次記録画面では投資先定義を変更しない。
- 比率合計は100.00%厳密一致を必須とする。
- 配分金額は1円単位で算出。
- 端数は最大比率の投資先へ配分。
- 比率変更は次の未確定月から適用する。
- 提案スタンスは中立とし、配分は設定比率に基づき機械的に算出する。
- **受入基準**:
- 比率合計が100.00%でない場合はエラーになること。
- 端数処理後の配分額合計が投資可能額と一致すること。
- 比率変更後、次の未確定月から新比率で配分されること。
- 設定画面で追加または編集した投資先配分が月次記録画面の投資先一覧へ反映されること。
- 一覧一括編集モーダル内で複数投資先をまとめて調整し、合計比率100.00%の状態で保存できること。

#### FR-206: 計算結果表示
- **説明**: 投資計算の結果と根拠値を表示する。
- **入力**: 投資可能額、配分額、予測月次収入、予測月次支出、累積収入、累積支出、余力資産、平均対象月数。
- **出力**: 計算結果表示データ。
- **ビジネスルール**:
- 表示項目は「投資可能額」「予測月次収入」「予測月次支出」「投資先ごとの配分額」「根拠値（累積収入・累積支出・余力資産）」「採用されたライフプランイベント」とする。
- ダッシュボードでは、投資先ごとの実績構成比、理想比率、差分、最新確定月の評価額を表示する。
- 平均対象月数を確認できること。
- 計算不可時は不足項目を箇条書きで表示する。
- **受入基準**:
- 表示項目と根拠値が同一画面で確認できること。
- 根拠値と結果の整合が取れていること。
- 計算不可時に不足項目の箇条書きが表示されること。
- 予測月次収入・予測月次支出・平均対象月数が確認できること。
- 採用されたライフプランイベントが確認できること。
- 各投資先の実績構成比、理想比率、差分、最新確定月の評価額が確認できること。

### 3.4 可視化
#### FR-301: 折れ線グラフ表示
- **説明**: 資産・収支の推移を折れ線グラフで表示する。
- **入力**: 表示対象、表示期間プリセット。
- **出力**: 折れ線グラフ。
- **ビジネスルール**:
- 表示対象は単一対象切替方式。
- ダッシュボードの数値サマリーと投資情報は最新の確定済み月を基準とする。
- 期間プリセットは `3か月/6か月/1年/全期間`。
- 初期表示期間は6か月とする。
- 折れ線グラフは確定済み月のみを表示対象とする。
- 折れ線グラフの選択対象には、通常資産に加えて各投資先の評価額推移も含める。
- ダッシュボード先頭に「投資可能額カード＋資産推移グラフ」を同時表示する。
- **受入基準**:
- 期間切替で再描画されること。
- 対象切替で表示系列が切り替わること。
- 初期表示時に6か月期間で表示されること。
- 最新の未確定月が存在しても、ダッシュボードの数値サマリーと投資情報は最新確定月を基準に表示されること。
- 未確定月が折れ線グラフに表示されないこと。
- ダッシュボード先頭に投資可能額カードと資産推移グラフが表示されること。

#### FR-302: 円グラフ表示
- **説明**: 最新確定月の投資評価額構成比を円グラフで表示する。
- **入力**: 最新確定月の投資先評価額、投資先名。
- **出力**: 円グラフ。
- **ビジネスルール**:
- 表示比率は `各投資先評価額 / 最新確定月の投資評価額合計` で算出する。
- 円グラフは理想比率ではなく、実績構成比を表示する。
- 投資評価額合計が0の場合は空状態を表示する。
- **受入基準**:
- 最新確定月の投資評価額構成比が円グラフで表示されること。
- 投資評価額合計が0の場合に円グラフではなく空状態が表示されること。

## 4. 非機能要件
### 4.1 パフォーマンス
- ダッシュボード初回表示は3秒以内を目標とする。
- 測定条件は通常データ量・通常回線とする。

### 4.2 セキュリティ
- 認証はメール+パスワード方式とする。
- パスワードは8文字以上かつ英数字混在を必須とする。
- メールによるパスワード再設定を提供する。

### 4.3 可用性
- 月間可用性 99.0% を目標とする。
- 可用性算定では計画メンテナンス時間を除外する。

### 4.4 スケーラビリティ
- v1では同時10ユーザー程度の利用を想定する。
- 同時10ユーザーは「同時に画面操作中のアクティブユーザー数」を指す。

## 5. 制約事項
- 金額は日本円（JPY）単一通貨で扱う。
- 価格変動資産の評価時点は月末とする。
- 月次境界はJSTの月初〜月末とする。
- 資産はカテゴリではなく名称単位で管理する。
- 投資先の追加、編集、削除および配分比率設定は設定画面で行う。
- 外部サービス連携（銀行/QR/カード/証券）はv1未対応で、手入力のみとする。
- v1で未対応: 月次リマインド通知、データエクスポート、バックアップ機能。
- 将来収支予測は確定済み月の直近6か月平均で自動算出し、手動入力や手動補正は行わない。
- 収入予測には賞与・臨時収入を含める。
- 支出予測は確定済み月の支出実績を平均し、過去の突発支出を自動除外しない。
- 将来の大きな支出はライフプランイベントとして別管理し、累積支出へ加算する。
- 設定変更時の再計算対象は未確定月のみとする。
- 期限超過月は対象月一覧＋警告バナーで通知する。
- 資産の追加/削除/復活の表示反映は、最新の確定月を境界に未確定月側へ適用する。
- 変更履歴（監査ログ）は保持しない。
- APIの具体I/Fは `API_SPECIFICATION.md` で定義する。
- 画面項目・UI挙動は `SCREEN_DESIGN.md` で定義する。

## 6. 用語集
| 用語 | 定義 |
|---|---|
| 月次確定 | ユーザーが月次データを手動で確定し、当該月を編集不可にする操作。 |
| 未確定月 | まだ月次確定されておらず、入力・補正が可能な月。 |
| 余力資産 | 累積収入から累積支出と生活防衛資金を差し引いた値。 |
| 投資可能額 | 余力資産を対象月までの月数で割って算出する毎月投資可能な金額。 |
| 予測月次収入 | 確定済み直近6か月の収入実績平均から自動算出した、将来計算用の月次収入。 |
| 予測月次支出 | 確定済み直近6か月の支出実績平均から自動算出した、将来計算用の通常月次支出。 |
| 資産 | 名称単位で管理する月次入力対象。預金や負債表現を含む。 |
| 投資先 | 設定画面で定義し、配分比率と月次投資評価額の対象になる単位。 |
| 月次投資評価額 | 月次記録画面で入力する、各投資先に対するその月の評価額。 |
| 有効資産 | 当月の入力項目として表示される状態の資産。 |
| 削除済み資産 | 入力項目には表示しないが、確定月までの履歴として保持される資産。 |
| 資産復活 | 削除済み資産を再び有効資産に戻し、最新の確定月の次の未確定月から再表示する操作。 |
